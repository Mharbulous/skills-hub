# Skills-hub

## Setup

Skills-hub's Cowork plugin is installed from the public Git repository:

```text
https://github.com/Mharbulous/skills-hub.git
```

### Claude Cowork

The preferred Cowork path is the Git repository marketplace:

```text
Customize > Plugins > Add marketplace > Add from a repository
https://github.com/Mharbulous/skills-hub.git
```

This clones the committed root marketplace at `.claude-plugin/marketplace.json`,
which points to `./plugins/skills-hub`.

When Cowork presents the `skills-hub` plugin card, install it. A new Cowork chat
should expose:

```text
/skills-hub
```

The plugin delivers the local `/skills-hub` control panel through Cowork's
plugin channel. Its inventory, install, and update flows use only the public
GitHub repository by default: the manager downloads the repo archive from
`codeload.github.com`, reads `public/skills`, and builds full Cowork `.skill`
packages locally before presenting them to Cowork.

No Firebase Hosting or signed static catalog is required for the `/skills-hub`
Cowork path.

### Claude Code and Codex

From this repo, run one of the legacy static installers:

```bash
SKILLS_BASE_URL="https://mharbulous.github.io/skills-hub" ./bootstrap/claude-setup.sh
SKILLS_BASE_URL="https://mharbulous.github.io/skills-hub" ./bootstrap/codex-setup.sh
```

`--full` is accepted as a compatibility alias.

Skills are edited in this repo under `public/skills`. `build/build_index.py`
still exists for legacy static artifact generation and plugin-tree generation,
but the Cowork `/skills-hub` manager does not depend on those hosted artifacts.

## Legacy Static Artifact Contract

These URLs are generated by `build/build_index.py` for older static-hosted
installers and wrappers. The Cowork `/skills-hub` manager does not require them.

Stable skill URLs:

```text
/{harness}/skills/<skill>/SKILL.md
/{harness}/skills/<skill>/<relative-path>
/{harness}/skills/<skill>.tar.gz
/{harness}/skills.tar.gz
/{harness}/stubs/<skill>/SKILL.md
/{harness}/skill-stubs.tar.gz
/{harness}/managed-skills.txt
/cowork/skill-packages/<skill>.skill
/cowork/skill-packages/<skill>.skill.b64.txt
/cowork/skill-packages/packages.json
/cowork/skill-packages/packages.json.sig
/cowork/install.json
/cowork/install.json.sig
/.claude-plugin/marketplace.json
/cowork/plugins/skills-hub/.claude-plugin/plugin.json
/cowork/plugins/skills-hub/skills/skills-hub/SKILL.md
/cowork/plugins/skills-hub/skills/skills-hub/<relative-path>
/index.json
/manifest.json
/manifest.json.sig
```

`harness` is `claude`, `codex`, or `cowork`.

Git repository marketplace artifacts are committed outside `public/`:

```text
.claude-plugin/marketplace.json
plugins/skills-hub/.claude-plugin/plugin.json
plugins/skills-hub/skills/skills-hub/SKILL.md
plugins/skills-hub/skills/skills-hub/<relative-path>
```

`index.json` remains schema v2 for browsing/back-compat. The optional
`manifest.json` schema v3 contains `generated_at`, `max_age_seconds`, the skill
catalog, and `sha256` plus `size` for every published file except the manifest
and signature.

Do not add dot-prefixed files to skills. Firebase Hosting ignores `**/.*`, so
the builder excludes dot-prefixed paths from `public/` and warns when it sees
them.

## Legacy Static Verification Model

The committed trust anchor is `bootstrap/skills_hub_allowed_signers`. The
private key is not committed; it lives in the GitHub secret
`SKILLS_HUB_SIGNING_KEY` and in Brahm's local signing-key backup for manual
deploys.

Verification is mechanical:

1. Download `manifest.json` and `manifest.json.sig`.
2. Verify the signature with `ssh-keygen -Y verify`.
3. Download the selected artifact into a temp file.
4. Verify that exact file's size and SHA-256 against the signed manifest.
5. Extract or read only the verified local bytes.

On signature, hash, size, freshness, or tar validation failure, installers and
resolvers fail closed and leave the previous install/cache intact.

## Consuming

Claude Cowork should install the Git repository marketplace:

```text
https://github.com/Mharbulous/skills-hub.git
```

The `/skills-hub` manager defaults to `Mharbulous/skills-hub@main`, downloads
the public GitHub archive, reads `public/skills`, and builds a full Cowork
`.skill` package locally for install/update.

Claude Code and Codex can still use the legacy static installers in
`bootstrap/` when a static host is available.

## Override Semantics

- Frontmatter keys in `overrides/<harness>.md` replace canonical keys.
- The generated `name` is always the canonical directory name.
- A non-empty override body is appended to the canonical body.
- No override file means that harness gets the canonical skill unchanged.

Run locally:

```bash
python build/build_index.py
```

For a manual legacy static deploy, build and deploy:

```powershell
python build\build_index.py
npx.cmd firebase-tools deploy --only hosting --project skills-hub
```

Signing static artifacts is optional legacy behavior via
`python build\build_index.py --require-signature`.

## Security Model

The repo is public. Never put secrets, client data, private tokens, or
environment credentials in skills or bundled subfiles.

The Cowork `/skills-hub` path trusts the public GitHub repository contents on
the selected ref. It protects against accidental package corruption by building
the `.skill` zip locally from the downloaded archive, but it does not provide a
signature chain. Guard repo write access like production code review access.

Legacy static signing can still protect hosted artifacts for consumers that
choose to use that path, but it is no longer required for Cowork `/skills-hub`
inventory, install, or update.
