# Skills-hub — Agent guidance

## Build

All marketplace JSONs, plugin.json files, and `public/cowork/` artifacts are
generated by `build/build_index.py`. Do not edit them directly. The canonical
skill definitions live in `public/skills/<name>/SKILL.md`.

Test suite: `python -m pytest tests` (not bare `python -m pytest` — the root
has generated directories that produce duplicate test module names).

## Plugin Staleness Debugging Rule

**Do not modify the repo to fix Cowork plugin staleness until you have ruled
out all three client-side causes below.** In every confirmed incident the repo
has been correct while one of these client-side states was the actual cause.

### Cause 1 — Saved skill overrides plugin (most common)

A `skills-hub` `.skill` previously imported into Cowork ("Save skill") takes
precedence over the plugin-delivered version. This is documented in
`plugins/skills-hub/skills/skills-hub/SKILL.md` lines 137–140.

Diagnostic: ask the user to check Cowork's user-skills list for a `skills-hub`
entry separate from the plugin card. If found, delete it.

### Cause 2 — URL marketplace instead of Git repository marketplace

Cowork cannot resolve relative plugin sources from a URL-based marketplace.
If the user added `mharbulous.github.io/skills-hub` or `skills-hub.web.app` as
a marketplace URL, the plugin source (`./plugins/skills-hub`) is unreachable
and Cowork falls back to a cached version.

Diagnostic: Customize > Plugins — check whether the marketplace entry is a URL
or a Git repository. The correct entry is:

```
https://github.com/Mharbulous/skills-hub.git
```

added via **Add from a repository**, not as a URL.

### Cause 3 — Cowork AppData plugin cache not purged on uninstall

Uninstalling the plugin via the UI may leave the old clone in Cowork's AppData
cache directory. Reinstalling reuses the cache.

Diagnostic: after uninstalling, check Cowork's AppData plugin cache for a
stale `skills-hub` clone. Delete it before reinstalling.

## What does not fix staleness

- Bumping `PLUGIN_VERSION` in `build/build_index.py`
- Regenerating marketplace JSONs
- Committing and pushing new plugin tree artifacts

These actions are irrelevant while any of the three client-side states above
are active.
